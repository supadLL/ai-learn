# Agent 是什么

Agent 是能根据目标和环境状态持续行动的 AI 系统。它通常包含观察、思考、行动、反馈和停止条件。普通聊天机器人只回答一次，Agent 会根据中间结果继续决定下一步。

Agent 的核心不是拟人化，而是闭环。它能看到当前状态，选择工具，执行动作，读取结果，再判断是否完成。这个循环让它适合处理步骤不固定的任务。

例如“帮我查资料并写报告”可能需要搜索、筛选、阅读、总结、补查、生成结构和引用来源。每一步的下一步都取决于上一步结果，这就是 Agent 更有价值的地方。

## Agent 和普通工作流的区别

普通工作流的步骤固定：先 A，再 B，再 C。Agent 的步骤可变：先观察结果，再决定下一步。固定流程更稳定、便宜、可控。Agent 更灵活，但成本和风险更高。

如果任务可以明确写成规则和分支，优先用工作流。如果任务需要在开放环境中探索、诊断、补充信息、动态规划，才考虑 Agent。

判断标准是：下一步是否依赖模型理解中间结果。如果不依赖，就不一定需要 Agent。

## Observe-Think-Act

典型 Agent 循环可以写成 Observe-Think-Act。Observe 是读取当前信息，例如用户目标、已有资料、工具返回和任务状态。Think 是判断下一步。Act 是调用工具或输出结果。

实际产品不一定要展示模型完整思考过程，但需要记录每一步行动：为什么调用这个工具，传了什么参数，得到什么结果，是否继续。这样用户和开发者才能理解 Agent 在做什么。

Agent 循环必须有停止条件。没有停止条件会导致无限搜索、无限重试或成本失控。

# 工具和记忆

Agent 的能力来自工具。工具可以让它搜索文档、读文件、查数据库、调用 API、写草稿、运行测试或创建任务。工具越强，越需要权限和审计。

记忆分短期和长期。短期记忆是当前上下文，长期记忆可以是数据库、向量库、用户偏好、历史任务和总结。记忆不是越多越好，错误记忆会污染后续任务。

写入长期记忆要有规则：什么值得记，什么时候记，用户是否允许删除，记忆是否需要过期。否则记忆会变成噪声仓库。

## 计划与执行

复杂 Agent 可以先生成计划，再逐步执行。计划让过程更可解释，也方便用户确认。但计划不能太死，因为中间结果可能改变方向。

一个好计划应该包含目标、步骤、每步输出、可用工具、停止条件和失败处理。例如“收集三份资料，提取共同结论，列出分歧，生成报告草稿”。每步都应该能被检查。

计划执行过程中要允许修正。发现资料不足时可以补查，发现目标不清时可以问用户，发现工具失败时可以降级。

## 多 Agent 协作

多 Agent 是把不同角色拆开，例如研究员、审稿人、执行者、规划者。它可以提高分工清晰度，但也会增加通信成本和错误传播。

很多任务不需要多 Agent。一个 Agent 加一个审查步骤就足够。只有当任务天然需要不同视角、不同工具权限或独立评审时，多 Agent 才值得引入。

多 Agent 系统要明确谁有最终决策权，谁能执行工具，谁只负责建议。否则角色之间可能互相放大错误。

# Agent 的风险

Agent 风险来自循环、工具、权限和目标误解。它可能重复无效操作，调用错误工具，执行用户未授权动作，或者为了完成目标绕过边界。

因此 Agent 必须有预算：最大步数、最大时间、最大成本、最大工具调用次数。还要有高风险动作确认、权限隔离、日志记录和人工接管。

Agent 的失败不一定是模型“笨”。也可能是目标不清、工具描述不清、返回格式混乱、停止条件缺失或环境权限错误。

## 安全设计

安全 Agent 的默认策略应该是最小权限。只给它完成任务所需的工具和数据。读取工具和写入工具分开，高风险工具默认关闭或需要确认。

代码执行、文件删除、支付、发邮件、修改配置、访问敏感数据，都要有沙箱或审批。模型不能靠一句提示词获得无限权限。

日志要记录行动链路。出问题时，需要能回答：谁发起任务，Agent 看到了什么，调用了什么工具，产生了什么结果，用户是否确认。

## 用户体验

Agent 产品要让用户看到进展。用户需要知道它正在做什么、卡在哪里、下一步是什么、是否需要确认。完全黑盒的 Agent 会让用户不信任。

好的体验包括：任务时间线、当前步骤、可暂停、可取消、危险动作确认、失败原因解释、最终结果引用和可编辑草稿。

Agent 不应该假装永远能完成任务。遇到信息不足、权限不足或工具失败时，应该清楚说明，并请求用户补充或选择降级路径。

# 什么时候适合 Agent

适合 Agent 的任务通常具备三个特征：目标明确但路径不固定，需要外部工具，能够根据中间结果调整。比如代码库排障、竞争情报收集、复杂文档问答、自动化测试修复、跨系统工单处理。

不适合 Agent 的任务包括：固定表单抽取、简单问答、确定规则判断、单次摘要、无需工具的文本改写。这些任务用普通 LLM 调用或工作流更稳。

学习 Agent 时，建议先做“只读 Agent”，比如只允许搜索和总结。理解循环和停止条件后，再加入写入动作和人工确认。
